# 01 — Спецификация продукта

## Назначение

Внутренний AI-ассистент: отвечает на вопросы сотрудников только на основе загруженных
администратором документов (PDF/DOCX/XLSX). Цели: точность, проверяемость (источники),
минимум галлюцинаций, минимум затрат на LLM.

## Зафиксированные решения (интервью с заказчиком, июль 2026)

| Вопрос | Решение |
|---|---|
| Пользователи | Сотрудники компании (внутренний инструмент) |
| Язык | Армянский — UI, документы, вопросы, ответы |
| Доступ к базе | Общая для всех, без групп доступа (v1) |
| Объём базы | До 100 документов |
| PDF | Только цифровые, OCR не требуется |
| Частота обновлений | Еженедельно |
| Тестовый датасет | Нет → генерируем синтетический (см. 05-EVALUATION.md) |
| Спорная релевантность | Отвечать с пометкой о неуверенности (двухпороговая схема) |
| Контекст диалога | Обязателен (query rewriting) |
| Внешние LLM API | Разрешены |
| Масштаб | 200–1000 сотрудников |
| Развёртывание | Не решено → закладываем 1 сервер + Docker Compose |

## Роли

**Administrator** — админ-панель: клиенты (CRUD, блокировка), документы (загрузка,
замена, удаление, переиндексация, статус), логи, аналитика (запросы, популярные
вопросы, % отказов, токены, кэш).

**Client** — только чат: авторизация, диалог с AI, история, источники ответов.

## Пайплайн ответа

```
Вопрос пользователя
  → Auth (JWT) + Rate limit
  → Query Rewriting (лёгкая LLM: уточняющий вопрос + история → самостоятельный запрос)
  → Hybrid Search (Qdrant dense + BM25) → RRF-слияние
  → Reranker (bge-reranker-v2-m3, top-20 → top-5)
  → Guardrail по score:
      score < T_low   → отказ БЕЗ вызова LLM: «Ցավոք, ձեր հարցի վերաբերյալ
                        տեղեկատվությունը գիտելիքների բազայում բացակայում է»
                        + подсказка переформулировать
      T_low…T_high    → вызов LLM + пометка неуверенности в ответе
      score ≥ T_high  → обычный вызов LLM
  → Context Builder (чанки + метаданные источников)
  → LLM (system prompt: отвечай ТОЛЬКО из контекста, на армянском)
  → Grounding-проверка (ответ опирается на чанки; для уверенного режима — обязательна)
  → Ответ + источники (документ, страница / лист+строка Excel)
```

Пороги T_low / T_high — в конфиге; калибруются на синтетическом датасете (05-EVALUATION.md).

## Ингестия документов

```
Файл → валидация формата/размера → S3 → задача BullMQ →
Docling (структура, таблицы) → semantic chunking →
embeddings (батчами) → Qdrant upsert + BM25 index → статус "готов"
```

- **Excel:** одна строка = один чанк, в текст чанка включаются заголовки колонок
  и имя листа («Թերթ: Սակագներ | Նահանգ: Տեխաս | Վճար: $120»), иначе строка
  без контекста бесполезна для поиска.
- **Замена документа:** новая версия индексируется в теневую коллекцию → атомарное
  переключение alias → удаление старых чанков → инвалидация кэша ответов по doc_id.
  Пользователи никогда не видят смесь версий.

## История чата

Хранится в PostgreSQL per-user. В контекст LLM попадают последние N сообщений;
при превышении бюджета токенов старые сообщения суммируются лёгкой моделью.

## Кэширование

- Redis: точное совпадение нормализованного вопроса → готовый ответ (TTL 7 дней,
  инвалидация при обновлении документов).
- Семантический кэш — v2, не в MVP.

## Безопасность

JWT (access 15 мин + refresh), RBAC-guard на каждом эндпоинте, bcrypt/argon2,
HTTPS, rate limiting (per-user: 20 вопросов/мин), audit_log действий админа,
CORS allowlist, Helmet.

## Явно вне рамок v1

Группы доступа к документам (схема готова — поле nullable), семантический кэш,
интеграции Slack/Teams/SSO, мобильное приложение, мультиязычность UI.
