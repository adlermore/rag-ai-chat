# 05 — Evaluation: качество ответов без готового датасета

Заказчик подтвердил: эталонного списка вопросов нет и составлять его некому.
Поэтому датасет генерируется автоматически из самих документов, а человек
только просматривает результат.

## Генерация синтетического датасета

1. Для каждого документа берём случайные 15–25% чанков.
2. Для каждого чанка LLM генерирует (на армянском):
   - 1 фактический вопрос, ответ на который содержится в чанке;
   - эталонный краткий ответ;
   - ссылку на источник (doc_id, page/row) — берётся из метаданных чанка.
3. Отдельно генерируем 30–40 **вопросов-ловушек**: по тематике компании, но ответа
   в документах заведомо нет. Правильное поведение системы — отказ.
4. Итог: 150–200 записей `{question, expected_answer, source, must_refuse: bool}`.

Просмотр: страница в админке (Фаза 4) — вопросы листаются, admin жмёт
approve/reject; 1–2 часа работы. Rejected исключаются из метрик.

## Метрики

| Метрика | Формула | Цель |
|---|---|---|
| Retrieval recall@5 | целевой чанк в top-5 после rerank | ≥ 0.85 |
| Answer correctness | LLM-judge: ответ соответствует эталону (шкала 1–5) | ≥ 4.0 средн. |
| Groundedness | LLM-judge: все утверждения ответа есть в контексте | ≥ 0.95 |
| False refusal rate | отказ при must_refuse=false | ≤ 5% |
| False answer rate | ответ при must_refuse=true | ≤ 3% |
| Latency p95 | полный пайплайн до первого токена | ≤ 4 c |

## Калибровка порогов T_low / T_high

1. Прогнать весь датасет, записать rerank-score лучшего чанка каждого вопроса.
2. Построить распределения score для must_refuse=true и false.
3. T_low — точка, где false answer rate ≤ 3%; T_high — где ответы выше порога
   имеют correctness ≥ 4.5 без пометки.
4. Значения зафиксировать в env и в `04-ROADMAP.md` (Фаза 4).

## Регрессия

Прогон датасета — команда `pnpm eval` (скрипт в apps/api/scripts).
Запускать: после смены embedding-модели, промпта LLM, параметров чанкинга,
и раз в месяц по расписанию. Результаты — в таблицу `eval_runs`, дельты
видны в админке.

## Постоянный источник улучшений

Реальные вопросы с отказом или low-confidence попадают в отчёт «вопросы без
ответа» в Analytics — это прямая подсказка админу, какие документы догрузить.
Лучшие реальные вопросы можно одной кнопкой добавлять в eval-датасет.
