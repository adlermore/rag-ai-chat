## Фаза 3 — Чат и RAG-пайплайн (недели 3–4)

- [x] API (backend): hybrid search → RRF → rerank → двухпороговый guardrail →
      context builder → LLM (SSE-стрим) → сохранение message + sources.
      Проверено e2e: релевантный вопрос → high + источники; off-topic → refused
      без LLM. LLM-провайдер: openai/anthropic/**stub** (работает без ключа).
- [x] Query rewriting: follow-up переписывается LLM в самостоятельный вопрос
      с учётом истории (проверено: «Իսկ նվազագույնը…» → полный вопрос → верный
      ответ). Stub-провайдер — pass-through.
- [x] Redis-кэш ответов: нормализованный переписанный вопрос → ответ+источники;
      инвалидация версией kb:version при изменении документов. Кэш-хит ~0.1s
      против ~38s полного прогона; cached-флаг в message.
- [x] Токен-метрики: tokensIn/tokensOut из usage провайдера сохраняются в БД.
- [~] Остаток: grounding-проверка ответа (базовая — через промпт).
- [x] UI чата по `03-DESIGN-SYSTEM.md`: SSE-стриминг с курсором, цитатные чипы ⟨n⟩
      (клик→скролл к источнику), карточки источников, кромка уверенности, отказ
      как спокойная карточка, пустой чат с примерами, история чатов. CORS для SSE.
- [x] Хвосты UI: HoverCard-превью цитаты (документ + фрагмент чанка из новой
      колонки message_sources.snippet), мобильный Sheet-сайдбар.

> Для реальных ответов (не stub) нужен ключ LLM: `OPENAI_API_KEY` (с балансом)
> или `ANTHROPIC_API_KEY`. Retrieval бесплатен (bge-m3).
