# 04 — Roadmap (порядок работы для Claude Code)

Работать строго по фазам. Каждая фаза заканчивается работающим, проверяемым результатом.

## Фаза 0 — Проверка армянского языка (блокирующая, ~2–3 дня)

Цель: снять главный риск проекта до написания продукта.

- [ ] Скрипт: взять 3–5 реальных армянских документов заказчика, прогнать через
      Docling, собрать 50–80 чанков.
- [ ] Вручную (или LLM-ом с проверкой) составить 30 вопросов с известным целевым чанком.
- [ ] Замерить recall@10 для: `text-embedding-3-large` vs `bge-m3`.
- [ ] Замерить вклад BM25 (армянский стеммер) и reranker `bge-reranker-v2-m3`
      (порядок top-5 осмыслен?).
- [ ] Решение зафиксировать в `docs/02-ARCHITECTURE.md` (таблица «Армянский язык»).

**Критерий выхода:** hybrid+rerank даёт recall@5 ≥ 0.85 на тестовых вопросах.

## Фаза 1 — Каркас (неделя 1)

- [ ] Monorepo (pnpm workspaces): apps/web, apps/api, apps/ingest, packages/shared, packages/ui.
- [ ] docker-compose: postgres, redis, qdrant, minio.
- [ ] NestJS: auth (JWT+refresh), users, RBAC-guard, rate limiting, audit_log.
- [ ] Next.js: shadcn init по `03-DESIGN-SYSTEM.md` (токены, шрифты Noto Armenian,
      светлая+тёмная темы), layout, страница логина, i18n-словарь hy.
- [ ] Миграции всех таблиц из `02-ARCHITECTURE.md`.

## Фаза 2 — Ингестия документов (неделя 2)

- [ ] ingest-сервис: Docling-парсинг PDF/DOCX/XLSX, semantic chunking,
      Excel «строка+шапка+лист», embeddings батчами, Qdrant upsert, BM25-индекс.
- [ ] BullMQ-пайплайн со статусами; SSE прогресса в админку.
- [ ] Админка Documents: таблица, dropzone-загрузка, статусы, замена (теневая
      коллекция + alias switch), удаление, переиндексация. Все 4 состояния экрана.

## Фаза 3 — Чат и RAG-пайплайн (недели 3–4)

- [ ] API: query rewriting → hybrid search → RRF → rerank → двухпороговый guardrail →
      context builder → LLM (SSE-стрим) → grounding-проверка → сохранение
      message + sources + метрик.
- [ ] Кэш Redis (нормализованный вопрос → ответ) + инвалидация при обновлении документов.
- [ ] UI чата по `03-DESIGN-SYSTEM.md`: стриминг, цитатные чипы ⟨n⟩ с HoverCard,
      карточки источников, кромка уверенности, состояния отказа/неуверенности,
      пустой чат с примерами вопросов, история чатов, мобильная версия.

## Фаза 4 — Evaluation и калибровка (неделя 5)

- [ ] Генератор синтетического датасета (см. `05-EVALUATION.md`): 150–200 Q/A/source.
- [ ] Скрипт прогонов: recall, точность порогов, % ложных отказов / ложных ответов.
- [ ] Подобрать THRESHOLD_LOW / THRESHOLD_HIGH; зафиксировать в env и в этом файле.
- [ ] Мини-review датасета админом (страница в админке: approve/reject вопросов).

## Фаза 5 — Аналитика и полировка (неделя 6)

- [ ] Dashboard: клиенты, документы, вопросы/день, % отказов, % low-confidence,
      токены, cache hit rate.
- [ ] Популярные вопросы; экспорт CSV.
- [ ] Клиенты: CRUD + блокировка; Audit-журнал.
- [ ] e2e-тесты happy path; нагрузочный smoke (50 параллельных вопросов).
- [ ] Прогон интерфейса с носителем армянского: тексты, длины строк, переносы.

## Definition of Done (каждая задача)

Типы strict · валидация входа · RBAC · состояния loading/empty/error ·
строки в i18n · unit-тесты логики · токены дизайна, без инлайн-стилей.
