# CLAUDE.md — Enterprise RAG Assistant (внутренний AI-ассистент)

## Что это за проект

Корпоративный AI-ассистент, отвечающий на вопросы сотрудников **исключительно на основе
внутренних документов компании** (PDF, DOCX, XLSX). Один централизованный ассистент,
не конструктор агентов. Полная спецификация: `docs/01-SPEC.md`.

## Ключевые факты (зафиксированы с заказчиком — НЕ менять без согласования)

- **Язык продукта: армянский (հայերեն).** Весь UI, ответы AI, сообщения об ошибках — на армянском.
  Документы в базе знаний — на армянском.
- Пользователи: сотрудники компании, 200–1000 человек. Роли: `admin` и `client`.
- База знаний общая для всех клиентов, **без** разграничения доступа к документам (v1).
- Объём: до 100 документов, все PDF цифровые (OCR не нужен), обновление еженедельно.
- Внешние LLM API разрешены (OpenAI / Anthropic).
- Guardrails: **двухпороговая** схема уверенности (см. SPEC §Guardrails):
  ниже нижнего порога — отказ без вызова LLM; между порогами — ответ с пометкой
  о неуверенности; выше верхнего — обычный ответ.
- Диалог с контекстом обязателен → в пайплайне есть шаг **query rewriting**.
- Каждый ответ обязан содержать источники: документ, страница, для Excel — лист и строка.

## Стек

| Слой | Технология |
|---|---|
| Frontend | Next.js 15 (App Router), React 19, TypeScript strict, Tailwind CSS v4, **shadcn/ui** |
| Backend | NestJS + Fastify, TypeScript strict |
| Парсинг документов | Docling (Python-микросервис) |
| Embeddings | OpenAI `text-embedding-3-large` (мультиязычная; `-small` слаба на армянском) |
| Vector DB | Qdrant |
| Keyword search | BM25 (армянский анализатор/стемминг) |
| Reranker | `bge-reranker-v2-m3` (мультиязычный, self-hosted CPU) |
| LLM | GPT-4.1-mini / Claude Sonnet — конфигурируемо через env |
| БД / кэш / очереди | PostgreSQL 16, Redis, BullMQ |
| Хранилище файлов | S3-совместимое (MinIO локально) |

## Структура репозитория (monorepo, pnpm workspaces)

```
apps/
  web/        # Next.js: чат клиента + админ-панель
  api/        # NestJS: auth, documents, chat, analytics
  ingest/     # Python FastAPI: Docling, chunking, embeddings
packages/
  shared/     # общие типы TS (DTO, enum'ы)
  ui/         # обёртки shadcn-компонентов, дизайн-токены
docs/         # эта документация
docker/       # docker-compose: postgres, qdrant, redis, minio
```

## Правила разработки

1. **Читай `docs/03-DESIGN-SYSTEM.md` перед любой работой над UI.** Не изобретай
   стили — используй токены и паттерны оттуда. Все компоненты — через shadcn/ui.
2. TypeScript strict everywhere. Никаких `any` без комментария-обоснования.
3. Все строки UI — через i18n-словарь (`hy` — основной, ключи на английском).
   Хардкод армянского текста в JSX запрещён.
4. API-контракты описываются в `packages/shared` и используются обеими сторонами.
5. Каждый эндпоинт: Zod/class-validator валидация входа, RBAC-guard, rate limiting.
6. Тесты: unit для сервисов пайплайна (chunking, thresholds, rewriting) обязательны;
   e2e — happy path чата и загрузки документа.
7. Пороги, модели, размеры чанков — только через конфиг/env, не в коде.
8. Логи админ-действий — в отдельную таблицу `audit_log` (кто, что, когда).
9. Коммиты — Conventional Commits. Одна фича — одна ветка.

## Чего НЕ делать

- Не добавлять разграничение доступа к документам (v2, но поле `access_group_id`
  в схеме документов заложить nullable уже сейчас).
- Не подключать Kubernetes/микросервисный оверинжиниринг — целевая среда:
  один сервер + Docker Compose.
- Не вызывать LLM, если гибридный поиск ничего не нашёл выше нижнего порога.
- Не отвечать пользователю без блока источников.
